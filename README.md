# Локальное распознавание аудио через Whisper

Простое локальное приложение: добавляете аудиозапись или запускаете запись системного звука, получаете текст. Модель Whisper загружается при запуске приложения.

## macOS-приложение и DMG

Собрать обычное macOS-приложение:

```bash
./scripts/build_macos_app.sh
```

Результат:

```text
dist/Whisper Local.app
```

Собрать установочный DMG:

```bash
./scripts/create_macos_dmg.sh
```

Результат:

```text
dist/Whisper Local.dmg
```

DMG открывается как обычный macOS-установщик: перетащите `Whisper Local` по стрелке в `Applications`.

По умолчанию модель Whisper и Python-библиотеки не кладутся внутрь DMG. При первом запуске `Whisper Local.app` само создаст окружение, скачает Python-пакеты из `requirements.txt` и загрузит модель `medium`.

Данные версии `.app` хранятся здесь:

```text
~/Library/Application Support/Whisper Local
```

Там будут `.venv`, модели, настройки и записи по умолчанию.

Для локального использования можно запускать неподписанную/ad-hoc подписанную сборку. Для распространения другим людям без предупреждений Gatekeeper нужен Developer ID и notarization, то есть Apple Developer Program.

## Запуск через ярлык

В проекте есть файл `Whisper.command`. Его можно запускать двойным кликом.

Ярлык на рабочем столе можно создать командой:

```bash
./install_shortcut.sh
```

После запуска приложение откроет браузер с локальной страницей:

```text
http://127.0.0.1:7860
```

## Запуск из терминала

Самый простой вариант:

```bash
./run.sh
```

Или вручную:

```bash
python3.12 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python app.py
```

При первом запуске модель будет скачана в `models`. После этого приложение можно запускать локально без повторной загрузки модели.

## Установка и настройка BlackHole

Для записи того, что звучит в браузере/системе, macOS нужен виртуальный аудиовход. Бесплатный вариант: `BlackHole 2ch`.

Установка через Homebrew:

```bash
brew install --cask blackhole-2ch
```

После установки перезагрузите Mac.

Если Homebrew нет, скачайте установщик с сайта Existential Audio:

```text
https://existential.audio/blackhole/
```

Настройка для наушников:

1. Откройте `Applications -> Utilities -> Audio MIDI Setup`.
2. Нажмите `+` в левом нижнем углу.
3. Выберите `Create Multi-Output Device`.
4. Включите `BlackHole 2ch` и ваши наушники, например `WH-1000XM6`.
5. Если не хотите слышать звук из динамиков MacBook, выключите `Динамики MacBook Pro`.
6. Для наушников включите `Drift Correction`; `BlackHole 2ch` оставьте главным устройством.
7. В `System Settings -> Sound -> Output` выберите созданное `Multi-Output Device`.

Настройка в Whisper Local:

- Для видео/браузера и микрофона: `Режим записи` -> `Приходящий + микрофон`, `Приходящий звук` -> `BlackHole 2ch`, `Микрофон` -> нужный микрофон.
- Только видео/браузер: `Режим записи` -> `Только приходящий`, `Приходящий звук` -> `BlackHole 2ch`.
- Только микрофон: `Режим записи` -> `Только микрофон`.

Важно: если в списке входов есть `WH-1000XM6`, `AirPods` или другие наушники, это обычно микрофон гарнитуры, а не звук, который играет в наушниках. Звук наушников macOS напрямую как вход не отдает.

## Источники аудио

В приложении есть системная запись через CoreAudio. Она не зависит от Safari или Chrome.

Режимы:

- `Приходящий + микрофон` — пишет системный/виртуальный вход и микрофон вместе.
- `Только приходящий` — пишет только выбранный входящий звук.
- `Только микрофон` — пишет только выбранный микрофон.
- `Все входы` — пишет все доступные входные устройства CoreAudio одновременно.
- `Выбрать аудиофайл` — распознать готовую запись.

Для приходящего звука выберите виртуальный аудиовход, например `BlackHole 2ch`, `Loopback`, `Piezo`, `Soundflower` или похожее устройство. На этой машине приложение видит `BlackHole 2ch`.

Если нужно писать звук всего Mac, направьте системный вывод в виртуальное устройство или Multi-Output Device, а в приложении выберите это устройство как `Приходящий звук`.

Если названия устройств не обновились, нажмите `Обновить устройства`.

Режим `Все входы` не умеет читать обычные выходы, например наушники или динамики, если у них нет входных каналов. Для системного звука все равно нужен `BlackHole 2ch` или другой виртуальный вход.

Важно: если в списке входов есть `WH-1000XM6`, `AirPods` или другие наушники, это почти всегда микрофон гарнитуры, а не звук, который играет в наушниках. Звук наушников macOS напрямую как вход не отдает.

Если системная запись не стартует, проверьте разрешение macOS:

```text
System Settings -> Privacy & Security -> Microphone
```

Разрешение нужно выдать приложению, из которого запускается `Whisper.command` или `python`.

## Папка записи

По умолчанию аудио и текст сохраняются в папку `recordings` внутри проекта.

В интерфейсе можно:

- нажать `Выбрать` и указать папку через macOS-диалог;
- вписать путь вручную в поле `Папка записи`;
- выключить `Сохранять аудио и текст`, если нужен только вывод текста на экран.

Настройка папки сохраняется в `settings.json` и применяется после перезапуска.

## Язык распознавания

По умолчанию приложение распознает как `Русский`, чтобы Whisper не путал русскую речь с близкими языками при шумной записи или коротком фрагменте.

В интерфейсе можно переключить `Язык распознавания` на:

- `Русский` — лучший режим для русских видео и разговоров.
- `Авто` — Whisper сам определяет язык.
- `Английский`, `Украинский`, `Белорусский` — ручной выбор для отдельных записей.

Настройка языка сохраняется в `settings.json`.

## Настройки модели

По умолчанию используется более качественный режим: модель `medium`, CPU и `float32`.

Это лучше распознает речь, чем прежний быстрый режим `base + int8`, но работает медленнее и при первом запуске скачает модель крупнее.

Можно выбрать другую модель перед запуском:

```bash
WHISPER_MODEL=small python app.py
```

Чем больше модель, тем выше качество и тем медленнее обработка:

- `tiny`
- `base`
- `small`
- `medium`
- `large-v3`

Быстрый режим:

```bash
WHISPER_MODEL=base WHISPER_COMPUTE_TYPE=int8 ./run.sh
```

Качественный режим по умолчанию:

```bash
WHISPER_MODEL=medium WHISPER_COMPUTE_TYPE=float32 ./run.sh
```

Еще выше качество, но намного медленнее:

```bash
WHISPER_MODEL=large-v3 WHISPER_COMPUTE_TYPE=float32 ./run.sh
```

## Форматы аудио

Поддерживаются распространенные аудиофайлы: `mp3`, `wav`, `m4a`, `ogg`, `flac`, `aac`, `aiff`, `webm`.
