# Record-Whisper

Локальное macOS-приложение для перевода аудио в текст через Whisper. Можно распознавать готовые аудиофайлы, микрофон, системный звук через BlackHole и смешанный режим “браузер + микрофон”.

Приложение работает локально: интерфейс открывается в окне macOS, а модель Whisper загружается на компьютер при первом запуске.

## Скачать

Готовый установочный пакет:

[Скачать Record-Whisper для macOS](packages/Record-Whisper-macOS.dmg)

После открытия DMG перетащите `Record-Whisper` по стрелке в `Applications`.

При первом запуске нужен интернет: приложение скачает модель Whisper `medium`. Это может занять несколько минут. Python runtime и Python-пакеты уже лежат внутри `.app`.

## Возможности

- Распознавание аудиофайлов: `mp3`, `wav`, `m4a`, `ogg`, `flac`, `aac`, `aiff`, `webm`.
- Запись микрофона.
- Запись системного/браузерного звука через `BlackHole 2ch`, `Loopback`, `Soundflower` или похожий виртуальный вход.
- Режим `Приходящий + микрофон` для звонков, видео, уроков и подкастов.
- Выбор папки сохранения.
- Сохранение аудио и текста рядом.
- Выбор языка распознавания. По умолчанию стоит `Русский`, чтобы Whisper не путал русскую речь с близкими языками.
- Кнопка выключения внутреннего сервера из интерфейса.
- Сборка красивого DMG-установщика со стрелкой `Record-Whisper -> Applications`.

## Требования

- macOS 13 или новее.
- Интернет при первом запуске для скачивания модели Whisper.
- Для записи системного звука: `BlackHole 2ch` или другой виртуальный аудиодрайвер.

## Быстрый Старт

1. Скачайте [Record-Whisper-macOS.dmg](packages/Record-Whisper-macOS.dmg).
2. Откройте DMG.
3. Перетащите `Record-Whisper` в `Applications`.
4. Запустите `Record-Whisper` из `Applications`.
5. При первом запуске дождитесь скачивания модели.

Если macOS пишет, что приложение от неизвестного разработчика:

1. Нажмите правой кнопкой по `Record-Whisper`.
2. Выберите `Открыть`.
3. Подтвердите `Открыть`.

Для локального использования подпись Apple Developer не нужна. Для распространения без предупреждений Gatekeeper нужна подпись Developer ID и notarization.

## Где Хранятся Данные

Версия `.app` хранит рабочие данные здесь:

```text
~/Library/Application Support/Record-Whisper
```

Там создаются:

- `models` с моделью Whisper;
- `settings.json` с настройками;
- `recordings` с записями по умолчанию.

Версия через `run.sh` хранит модели и записи внутри папки проекта.

## Настройка BlackHole

macOS не дает напрямую записывать звук, который играет в наушниках или динамиках. Для этого нужен виртуальный аудиовход. Бесплатный вариант: `BlackHole 2ch`.

Установка через Homebrew:

```bash
brew install --cask blackhole-2ch
```

После установки обязательно перезагрузите Mac.

Если Homebrew нет, скачайте BlackHole вручную:

[https://existential.audio/blackhole/](https://existential.audio/blackhole/)

## BlackHole + Наушники

Чтобы одновременно слышать звук в наушниках и записывать его в Whisper:

1. Откройте `Applications -> Utilities -> Audio MIDI Setup`.
2. Нажмите `+` в левом нижнем углу.
3. Выберите `Create Multi-Output Device`.
4. Включите `BlackHole 2ch`.
5. Включите ваши наушники, например `WH-1000XM6`.
6. Если не хотите слышать звук из динамиков MacBook, выключите `Динамики MacBook Pro`.
7. Для наушников включите `Drift Correction`.
8. `BlackHole 2ch` оставьте главным устройством.
9. В `System Settings -> Sound -> Output` выберите созданное `Multi-Output Device`.

Настройка в Record-Whisper:

- `Режим записи`: `Приходящий + микрофон`
- `Приходящий звук`: `BlackHole 2ch`
- `Микрофон`: нужный микрофон или микрофон гарнитуры
- `Язык распознавания`: `Русский`

Для записи только браузера:

- `Режим записи`: `Только приходящий`
- `Приходящий звук`: `BlackHole 2ch`

Для записи только микрофона:

- `Режим записи`: `Только микрофон`
- `Микрофон`: нужное устройство

Важно: если в списке входов есть `WH-1000XM6`, `AirPods` или другие наушники, это обычно микрофон гарнитуры, а не звук, который играет в наушниках.

## Сборка Пакета

Собрать `.app`:

```bash
./scripts/build_macos_app.sh
```

Результат:

```text
dist/Record-Whisper.app
```

Собрать установочный DMG:

```bash
./scripts/create_macos_dmg.sh
```

Результат:

```text
dist/Record-Whisper.dmg
```

Скрипт DMG:

- собирает Swift-обертку с `WKWebView`;
- добавляет Python runtime, Python-пакеты и FastAPI backend внутрь `.app`;
- генерирует иконку приложения;
- генерирует фон установщика со стрелкой;
- создает DMG с раскладкой `Record-Whisper -> Applications`.

Python runtime и Python-пакеты кладутся внутрь `.app`. Модель Whisper не кладется в DMG и скачивается при первом запуске приложения.

## Запуск Без Пакета

Через ярлык:

```bash
./install_shortcut.sh
```

После этого можно запускать `Whisper.command` двойным кликом.

Через терминал:

```bash
./run.sh
```

Ручной запуск:

```bash
python3.12 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python app.py
```

Локальный адрес:

```text
http://127.0.0.1:7860
```

## Настройки Модели

По умолчанию:

- модель: `medium`;
- устройство: `cpu`;
- тип вычислений: `float32`;
- язык: `ru`.

Можно выбрать другую модель перед запуском:

```bash
WHISPER_MODEL=small ./run.sh
```

Варианты:

- `tiny`
- `base`
- `small`
- `medium`
- `large-v3`

Быстрый режим:

```bash
WHISPER_MODEL=base WHISPER_COMPUTE_TYPE=int8 ./run.sh
```

Максимально качественный, но медленный режим:

```bash
WHISPER_MODEL=large-v3 WHISPER_COMPUTE_TYPE=float32 ./run.sh
```
